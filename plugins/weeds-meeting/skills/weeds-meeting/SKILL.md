# WEEDS 회의록 작성 가이드

## 개요

이 스킬은 WEEDS 회사의 표준 회의록 형식에 맞춰 회의 내용을 정리하고 문서화합니다. 회의 참석자, 일시, 장소, 의제, 논의 내용 등을 입력받아 구조화된 회의록을 생성합니다.

## 회의록 작성 프로세스

### 1. 정보 수집

사용자로부터 다음 정보를 수집합니다:

**필수 정보:**
- 회의 주제 (회의명에 사용)
- 일시 (날짜, 시작/종료 시간)
- 주관 부서/팀
- 내부 참석자 (팀, 직급, 이름)
- 회의 장소
- 의제 (논의 주제)
- 회의 내용 및 결과

**선택 정보:**
- 외부 참석자 (있는 경우)

사용자가 제공한 정보가 불충분한 경우, 필요한 정보를 추가로 요청합니다.

### 2. 회의록 구조화

수집한 정보를 다음의 표준 형식으로 구조화합니다:

**제목 형식:**
```
회의명 : [YYMMDD 또는 간략한 날짜] [회의 주제]
```

**기본 정보 표:**

| 항목 | 내용 |
|------|------|
| ○ 주관부 | [부서/팀명] |
| ○ 일 시 : | YYYY-MM-DD(요일) HH시 MM분 ~ YYYY-MM-DD(요일) HH시 MM분 |
| ○ 장 소 : | [회의 장소] |
| ○ 내부참석자 : | [팀명] 팀: [직급] [이름], ...<br/>[다른 팀명] 팀: ... |
| ○ 외부참석자 | [있는 경우 기재, 없으면 빈 칸] |
| ○ 의 제 : | [회의 주제 및 안건] |

**회의결과 표:**

| 항목 | 내용 |
|------|------|
| ○ 회의결과 | |
| - 내용 | [회의 결과 및 결론 요약] |

### 3. 내용 작성 원칙

**간결성**
- 핵심 내용만 명확하게 기술
- 불필요한 세부사항은 생략
- 요점 중심으로 정리

**정확성**
- 참석자명, 직급, 소속팀 정확히 기재
- 일시와 장소는 정확하게 표기
- 결정사항과 논의사항 명확히 구분

**구조화**
- 표 형식을 준수하여 가독성 확보
- 여러 안건이 있는 경우 {A}, {B} 등으로 구분
- 팀별로 참석자를 그룹화

**요약**
- 회의결과는 장황하지 않게 핵심만 요약
- 액션 아이템이 있으면 명확히 기재
- 결론과 다음 단계를 명시

## 특수 사례 처리

### 복수 안건 처리
여러 안건이 있는 경우 다음과 같이 구분:

```
○ 의 제 : [ 카테고리 ]
          {A} : 첫 번째 안건
          {B} : 두 번째 안건
```

### 팀 표기 방법
내부참석자 표기는 두 가지 방식 모두 사용 가능:

**방식 1:**
```
AI 팀: 김혁 팀장, 손정민 BA
PE 팀: 이인범 BA
```

**방식 2:**
```
[기획] 지선구 팀장, 너클라 BE
[AI] 김혁 팀장
[PE] 이인범 BA
```

### 날짜/시간 형식
- 전체 형식: `2025-06-12(목) 13시 00분`
- 요일은 괄호 안에 한글로 표기
- 시작 시간과 종료 시간을 `~`로 연결

## 작성 흐름

1. 사용자로부터 회의 정보 수집
2. 부족한 정보가 있으면 추가 질문
3. 표준 형식에 맞춰 회의록 생성
4. 생성된 회의록을 파일로 제공 (docx 또는 markdown)

**참고:** 사용자가 회의 녹취록, 메모, 또는 비정형 텍스트를 제공하는 경우, 해당 내용을 분석하여 필요한 정보를 추출하고 표준 형식으로 변환합니다.
