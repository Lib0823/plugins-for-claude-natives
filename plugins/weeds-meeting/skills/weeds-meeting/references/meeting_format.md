# WEEDS 회의록 표준 형식

## 회의록 구조

회의록은 다음의 표 형식으로 작성됩니다.

### 기본 정보 표

| 항목 | 내용 |
|------|------|
| ○ 주관부 | [부서/팀명] |
| ○ 일 시 : | YYYY-MM-DD(요일) HH시 MM분 ~ YYYY-MM-DD(요일) HH시 MM분 |
| ○ 장 소 : | [회의 장소] |
| ○ 내부참석자 : | [팀명] 팀: [직급] [이름], [직급] [이름]<br/>[팀명] 팀: [직급] [이름] |
| ○ 외부참석자 | [있는 경우 기재] |
| ○ 의 제 : | [회의 주제 및 안건] |

### 회의결과 표

| 항목 | 내용 |
|------|------|
| ○ 회의결과 | |
| - 내용 | [회의 결과 및 결론 요약] |

## 필드별 작성 규칙

### 1. 회의명 (제목)
- 형식: `회의명 : [간략한 회의 주제]`
- 날짜 포함 시: `YYMMDD [주제]` 형식 사용
- 예시:
  - `회의명 : 20250612 HMAC 관리대장 시스템 설계 및 구현 방안 논의`
  - `회의명 : 251105 정제 화면정보 delete_yn 및 개인정보 자동 삭제에 대한 논의`

### 2. 주관부
- 회의를 주관하는 부서나 팀명 기재
- 예시: `AI / PE 팀`, `PE팀`

### 3. 일시
- 형식: `YYYY-MM-DD(요일) HH시 MM분 ~ YYYY-MM-DD(요일) HH시 MM분`
- 요일은 괄호 안에 한글로 표기
- 예시: `2025-06-12(목) 13시 00분 ~ 2025-06-12(목) 15시 00분`

### 4. 장소
- 회의실명이나 구체적인 장소 기재
- 예시: `4층 회의실`

### 5. 내부참석자
- 팀별로 구분하여 작성
- 형식: `[팀명] 팀: [직급] [이름], [직급] [이름]`
- 여러 팀인 경우 줄바꿈으로 구분
- 예시:
  ```
  AI 팀: 김혁 팀장, 손정민 BA
  PE 팀: 이인범 BA
  ```
  또는
  ```
  [기획] 지선구 팀장, 너클라 BE
  [AI] 김혁 팀장
  [PE] 이인범 BA
  ```

### 6. 외부참석자
- 외부 참석자가 있는 경우에만 기재
- 없으면 빈 칸으로 둠

### 7. 의제
- 회의의 주요 논의 주제 또는 안건 나열
- 여러 안건이 있는 경우 구분하여 작성
- 예시:
  - 단일 안건: `HMAC 관리대장 시스템 설계 및 구현 방안 논의`
  - 복수 안건: `[ 정제 ]`<br/>`{A} : 화면 정제 delete_yn 처리 방향에 대한 논의`<br/>`{B} : 개인정보 자동 삭제 기능에 대한 논의`

### 8. 회의결과 - 내용
- 회의에서 논의된 주요 내용과 결정사항을 요약
- 구체적이고 실행 가능한 내용으로 작성
- 필요시 항목별로 구분하여 작성

## 작성 시 주의사항

1. **간결성**: 핵심 내용만 명확하게 기술
2. **구조화**: 표 형식을 준수하여 가독성 확보
3. **정확성**: 참석자, 일시, 장소 등 기본 정보는 정확하게 기재
4. **요약**: 회의결과는 장황하지 않게 핵심만 요약
5. **팀 구분**: 내부참석자는 반드시 팀별로 구분하여 작성
